# 我的世界 AI 村庄直播客户端说明

这份说明给运行 Minecraft 客户端和 OBS 的局域网机器使用。服务端、AI、Mindcraft、控制台都运行在另一台服务器上。

## 当前连接信息

- Minecraft 服务器地址：`192.168.3.133:25565`
- 直播可视化页面：`http://192.168.3.133:3010/`
- 观察者账号名：`live`
- 推荐客户端版本：Minecraft Java，尽量与服务器版本一致，目前服务端日志显示 `1.21.11`
- 控制台不需要在客户端打开：控制台目前只在服务器本机 `127.0.0.1:4177` 可访问

注意：客户端机器不要使用 `127.0.0.1` 访问这些服务。`127.0.0.1` 在客户端机器上代表它自己，不代表服务器。

## 客户端进入服务器

1. 打开 Minecraft Java 启动器。
2. 使用用户名 `live` 启动游戏。
3. 进入多人游戏。
4. 添加服务器：`192.168.3.133:25565`。
5. 加入服务器后不要操作角色，等待服务端把 `live` 切成观察者并锁定某个 AI。

如果进入后没有自动跟随 AI，让服务器端操作人员点击控制台里的“live 看活跃 AI”，或在服务端执行：

```text
/gamemode spectator live
/spectate Ivy live
```

目标不一定必须是 `Ivy`，也可以换成 `Alex`、`Luna`、`Milo`、`Nova`。

## 推荐直播方式

### 方案 A：OBS 捕获 Minecraft 客户端

这是主直播画面，画质最好。

1. 打开 OBS。
2. 新建场景，例如“AI 村庄直播”。
3. 添加来源：游戏捕获。
4. 模式选择“捕获特定窗口”。
5. 窗口选择 Minecraft。
6. 分辨率建议 1920x1080，60fps 或 30fps 都可以。
7. Minecraft 里按一次 `F1` 可以隐藏界面，让画面更干净；如果想保留聊天和状态，就不要按 `F1`。

### 方案 B：OBS 添加网页可视化

可作为辅助信息层，用来展示 AI 状态、村庄进度、思考字幕等。

1. OBS 添加来源：浏览器。
2. URL 填：`http://192.168.3.133:3010/`。
3. 宽高建议：`1920 x 1080`。
4. 可以单独做一个场景，也可以叠在 Minecraft 画面旁边。

### 方案 C：OBS 添加直播状态覆盖层

这是专门叠在 Minecraft 主画面上的轻量覆盖层，会跟随服务器端 `live` 自动切换状态显示当前镜头目标。

1. OBS 添加来源：浏览器。
2. URL 填：`http://192.168.3.133:3010/obs?compact=1`。
3. 宽高建议：`900 x 260` 或按画面布局调整。
4. 放在画面底部或左下角。
5. 覆盖层会显示：`镜头：live → 当前AI｜自动轮换 30s`，并在切换目标时短暂高亮。
6. 如果想固定显示某个 AI 的信息，可使用：`http://192.168.3.133:3010/obs?agent=Luna&compact=1`。


### 方案 D：OBS 使用完整直播导播页

这是推荐的正式直播版式。页面左上是固定 16:9 的黑色主画面占位窗口，1920x1080 画布下大约是 1440x810；右侧约 1/4 是紧凑状态栏。OBS 里把 Minecraft 客户端/游戏捕获叠到左侧黑色窗口上，让观众主要看游戏画面。

1. OBS 添加来源：浏览器。
2. URL 填：`http://192.168.3.133:3010/studio`。
3. 宽高建议：`1920 x 1080`。
4. 把这个浏览器源作为底层场景。
5. 再添加 Minecraft 游戏捕获，放到左侧黑色占位窗口上方，大小对齐左侧 16:9 大窗口；1920x1080 场景中可先按 1440x810 摆放。
6. 右侧状态栏只保留：村长指令、世界状态、当前居民任务、公共建设和互动提示。
7. 左侧画面上方会显示当前镜头目标和世界模式，切换观察目标时会短暂高亮。

如果只想要一个轻量信息条，继续使用：`http://192.168.3.133:3010/obs?compact=1`。
## 避免画面变成菜单

如果切出 Minecraft 后直播画面变成游戏菜单，按下面处理：

1. 尽量用“窗口模式”或“无边框窗口”，不要用独占全屏。
2. OBS 使用“游戏捕获/捕获特定窗口”，不要只用显示器捕获。
3. 进入服务器后不要按 `Esc`。
4. 如果已经出现菜单，点回 Minecraft 窗口，按一次 `Esc` 回到游戏画面。
5. 在游戏里按 `F3 + P`，把“失去焦点时暂停”关闭。虽然多人服务器不会真正暂停，但这个设置可以减少切出时的异常菜单表现。
6. 最稳的方式是让 Minecraft 窗口一直保持前台，把 OBS 控制放到第二屏、手机远控或另一台机器上。

## 常见问题

### 连接不上服务器

检查客户端能否访问服务器 IP：

```text
192.168.3.133:25565
```

如果不行，确认两台机器在同一个局域网，Windows 防火墙允许 Java/Minecraft 服务端端口 `25565`。

### 打开 3DView 访问不到

不要访问：

```text
http://127.0.0.1:3002/
```

这是服务器本机地址，局域网客户端访问不到。请使用：

```text
http://192.168.3.133:3010/
```

### `live` 没有跟随 AI

说明 `live` 只是进服了，还没有被服务端切到旁观目标。通知服务器端操作人员执行“live 看活跃 AI”。

### OBS 黑屏

优先尝试：

1. OBS 和 Minecraft 使用同一块显卡运行。
2. OBS 来源改成“窗口捕获”或“显示器捕获”测试。
3. Minecraft 改成窗口模式。
4. 关闭光影或高风险渲染模组后再试。

## 直播时的分工

- 客户端机器：只负责运行 `live` 账号、Minecraft 画面和 OBS 推流。
- 服务器机器：负责 Minecraft 服务端、Mindcraft、AI 居民、村长、任务调度和观察者切换。
- 客户端不需要配置任何 API key，也不要打开控制台配置页。

## 推荐开播检查清单

开播前确认：

- `live` 已进入服务器。
- 服务端在线列表里能看到 `live`。
- 服务端已显示类似 `Now spectating Ivy` 的日志。
- OBS 正在捕获 Minecraft 客户端画面。
- 如需辅助面板，OBS 浏览器来源使用 `http://192.168.3.133:3010/`。
- Minecraft 画面没有停在菜单界面。
- 声音来源、码率、推流地址都已设置完成。

## 主播端 MCP 网关

服务器现在提供一个给主播端使用的受限 MCP 网关。它只开放直播镜头、村庄状态和宏观目标相关工具，不开放完整控制台配置、密钥、服务启停等高权限能力。网关需要 token，服务器端 token 存在 `data/streamer-mcp-token.txt`，不要提交到公开仓库。

- MCP SSE 地址：`http://192.168.3.133:4178/mcp/sse?token=<主播网关TOKEN>`
- Streamable MCP 地址：`http://192.168.3.133:4178/mcp?token=<主播网关TOKEN>`
- 健康检查：`http://192.168.3.133:4178/api/health?token=<主播网关TOKEN>`
- 主播状态 JSON：`http://192.168.3.133:4178/api/stream/status?token=<主播网关TOKEN>`

可用工具：

- `get_stream_status`：查看服务器、Mindcraft、在线居民、直播镜头和村庄摘要。
- `village_report`：获取基地、公共箱子、项目、资源和设施进度。
- `focus_live_observer`：切换 `live` 观察者到指定 AI 或 `auto`。
- `set_live_auto_switch`：开启/关闭自动轮换，或调整轮换间隔。
- `send_village_goal`：给村长/居民系统发送宏观目标。
- `send_agent_task`：给单个居民或所有在线 AI 发送任务。

示例指令：

```text
把直播镜头切到 Luna
开启 live 自动轮换，每 30 秒切一次
看看村庄当前建设进度
让村长今晚优先补光、整理公共箱子，并让居民回基地附近活动
```

如果主播端工具支持 MCP SSE，配置示例：

```json
{
  "mcp_servers": {
    "minecraft-ai-friend-streamer": {
      "transport": "sse",
      "url": "http://192.168.3.133:4178/mcp/sse?token=<主播网关TOKEN>"
    }
  }
}
```

## 直播可视化宏观信息

`http://192.168.3.133:3010/` 页面现在会显示更多村庄宏观信息，包括：

- 村长宏观指令
- 基地坐标和公共箱子坐标
- 当前直播观察目标和自动轮换状态
